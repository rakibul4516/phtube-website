Discuss the scope of var, let, and const.

var,let and const keyword is used for variable declaration.But variables declared with var,let and const based on different conditions.JavaScript has two main types of scope."Global scope" and "Block-level scope"."var" is a global scope variable on the other hand "let" and "const" is block-level scope variable.Here is the explain bellow:

Var:- "var" is a global scope variable.The global scope are accessible from anywhere in JavaScript code.If we declare a "var" variable outside of any function or block of code, it becomes a global variable.It is accessible from anywhere in your JavaScript code.We can reassign the value of this variable.

Let:- "let" is block-level scope, which means it is accessible into the nearest enclosing block. The block can be a function, an if else statement, a loop, or any other code block enclosed by curly braces { }.Value of this variable can reassigned and also can apply function for remove or add values.

Const:- Variables declared with "const" are block-scoped, only accessible within the block in which we are defined. The block can be a function, conditional statement, a loop, or any other code block enclosed by curly braces { } like 'let'.'const' are constant, which means values cannot be reassigned after initialized. Once assign a value to a const variable,cannot change that value.But we can apply function for remove or add values.

What do you mean by REST API?

REST APIs use standard HTTP methods to perform operations.It's used to communicate between clients and servers.Resources are represented in different formats such as JSON (JavaScript Object Notation) and XML (eXtensible Markup Language).Clients request to server for get data.If server has the data and request is right server will provide desired data other wise server will return an error message.REST APIs separates the client and server side for increse scalability and flexibility.

The most common HTTP methods used in REST APIs are:

  • GET: GET used for read-only operations.
  • POST: Create a new resource.
  • PATCH: Partially update a resource.
  • PUT: Update an existing resource or create it if it doesn't exist.
  • DELETE: Remove a resource.
  • HEAD: HEAD request to a URL for headers containing information.
